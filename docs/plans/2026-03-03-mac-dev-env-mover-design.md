# Mac Dev Env Mover v1 Design

## Goal
Recreate a personal developer environment from one Mac to another using a local export/import bundle, without cloning the entire machine.

## Architecture
- Native macOS SwiftUI app for operator workflow.
- Core logic isolated from UI so export/import/verify are testable without UI.
- Manifest-driven data contract (`schemaVersion=1.0.0`) to coordinate export/import/reporting.
- Process-based command execution with explicit executable + argument array.

## Components
- `SharedModels`: manifest schema types, restore phases, item kinds, reports.
- `Core`: preflight, machine info collection, manifest read/write/validate, backup naming, path normalization, structured logging, verify engine.
- `Exporters`: Homebrew, dotfiles allowlist, Git global, VS Code exporters + export coordinator.
- `Importers`: bundle validation, package/config/IDE restore, manual task generation, verify/report orchestration.
- `Reporting`: markdown report generation for export/import/verify.
- `App`: tabs for Overview, Export, Import, Reports.

## Data Flow
1. Export preflight checks environment and destination writability.
2. Exporters gather data/files and produce manifest items.
3. Restore plan is generated by phase ordering.
4. Manifest + reports + files are written to bundle.
5. Import validates bundle + preflight.
6. Import applies packages/config/IDE with backup-on-overwrite.
7. Verify runs and writes verify report.

## Error Handling
- Blocking preflight failures stop the current operation.
- Component-level failures are collected in reports and do not force global abort when safe to continue.
- Unsupported/secret/missing prerequisites are represented as manual tasks.

## Security/Policy
- Secret-like content is excluded by default and surfaced as manual transfer tasks.
- No cloud sync, account system, server, background agent, or daemon.
